Connect n8n to Dashboard

Follow these steps to wire your BuyerLeadQualifier workflow into this dashboard.

Step 1 — Host this dashboard on GitHub Pages

Upload this HTML file to a GitHub repo, enable GitHub Pages in Settings → Pages → Deploy from branch (main, /root). Your dashboard URL will be:
https://YOUR_USERNAME.github.io/REPO_NAME/BuyerLeadQualifier_Dashboard.html

1
Go to github.com → New Repository, name it e.g. kolkata-leads-dashboard, set it Public.
2
Upload this BuyerLeadQualifier_Dashboard.html file directly into the repo root.
3
Go to Settings → Pages → Source → Deploy from branch, select main and / (root), click Save.
4
Wait ~60 seconds. Your live URL appears at the top of the Pages settings page.

Step 2 — Set up a free Webhook receiver (JSONBin)

This dashboard reads lead data posted to a public JSONBin bin. It's free, no backend needed.

1
Go to jsonbin.io → Sign up free → Click "Create Bin".
2
Paste this starter JSON and save:
{"leads": [], "summary": {"total": 0, "hot": 0, "warm": 0, "cold": 0, "avg_score": "0"}}
3
Copy your Bin ID (looks like 64abc123...) and your Master Key from Account → API Keys.
4
Paste them below to configure the dashboard:

Step 3 — Update the n8n HTTP Request node

Replace the last node (HTTP Request — Post to Dashboard) in your n8n workflow with these settings:

1
Method: PUT
2
URL:
https://api.jsonbin.io/v3/b/YOUR_BIN_ID
3
Headers to add:
Content-Type: application/json X-Master-Key: YOUR_MASTER_KEY X-Bin-Versioning: false
4
Body (JSON):
={{ JSON.stringify({ leads: $json.all_leads, summary: $json.report_summary }) }}

Step 4 — Test the full pipeline

1
Run your n8n workflow manually from the Manual Trigger node.
2
Come back to this dashboard → click the Dashboard tab → click Refresh.
3
Your scored leads should appear in the table instantly 🎉
